# 核心概念

**Scalability (可扩展性)**
- **定义**: 系统在需求增加时增长和管理的能力。
- **属性**:
  - **Horizontal Scaling (横向扩展)**: 通过增加更多服务器到资源池中。
  - **Vertical Scaling (纵向扩展)**: 通过增加现有服务器的资源（CPU、内存、存储等）。这种方法会带来停机时间并且有一个上限。

**Reliability (可靠性)**
- **定义**: 系统在给定时间段内失败的概率。
- **属性**:
  - 一个分布式系统被认为是可靠的，如果它在一个或多个组件故障时仍能继续提供服务。
  - 通过组件和数据的冗余实现可靠性，消除单点故障。

**Availability (可用性)**
- **定义**: 系统在特定时间段内保持运行以执行其所需功能的时间。
- **属性**:
  - 通过系统在正常条件下保持运行的百分比来衡量。
  - 可靠的系统是可用的，但可用的系统不一定是可靠的（例如，一个有安全漏洞的系统在没有遭受安全攻击时是可用的）。

**Efficiency (效率)**
- **属性**:
  - **Latency (延迟)**: 响应时间，获取第一段数据的延迟。
  - **Bandwidth (带宽)**: 吞吐量，在给定时间内传输的数据量。

**Serviceability / Manageability (可服务性 / 可管理性)**
- **属性**:
  - 系统的操作和维护的简便性。
  - 系统修复或维护的简易性和速度。
